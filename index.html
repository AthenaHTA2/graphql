Query examples:
Query table 'user':
{
query {
user {
id
login
}
}
}

query for table 'user' that includes variables:
{
query {
user(where: { id: { _eq: 6 }}) {
id
login
}
}
}

To curl the API endpoint to see the result given by the server:
curl "https://((DOMAIN))/api/graphql-engine/v1/graphql" --data '{"query":"{user{id login}}"}'
with variables: curl "https://((DOMAIN))/api/graphql-engine/v1/graphql" --data '{"query":"{user(where:{id:{_eq:6}}){id login}}"}'
In graphQL the usage of arguments can be specified in the schema of the API.
As mentioned above, you can visit the docs for the graphQL endpoint in:
https://%28%28domain%29%29/graphiql

Example of nesting, using the result and user table :
{
result {
id
user {
id
login
}
}
}

we ask for the results id and users that are associated to the result,
requesting the users logins and ids.
The types of queries to be used, together or separately, are normal, nested and using arguments
